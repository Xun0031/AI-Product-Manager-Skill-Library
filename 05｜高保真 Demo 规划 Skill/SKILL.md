# 05｜高保真 Demo 规划 Skill

> 使用前请先对输入信息进行匿名化处理，不要粘贴客户机密、个人隐私、内部链接、账号密钥、未授权的项目原文或真实业务数据。示例中的组织、人物、项目和数据均为虚构或脱敏内容。

## 1. Skill 定位

将定位、流程、产品架构和方案主线转化为 Demo 制作蓝图。输出是演示规划，不是最终页面、原型或代码。

## 2. 关于“高保真”的定义

这里的高保真指业务链路完整、路径清晰、范围合理、样例数据可信、能力与承诺边界清楚，不特指视觉层面的精细程度。

## 3. 适用场景

- 方案沟通、需求确认或内部评审演示
- 多角色 B 端平台的闭环展示
- 原型或前端演示制作前规划
- AI 辅助能力的边界化展示

## 4. 不适用场景

- 前置定位、流程或架构未明确
- 直接生成视觉稿、原型或代码
- 详细需求或技术方案

## 5. 本 Skill 可以输出什么

- 演示目标、对象与场景
- 主线、角色、端口和页面范围
- 页面优先级、路径和样例数据
- 讲解重点、制作深度和任务拆解
- AI、演示状态与承诺边界

## 6. 本 Skill 不直接输出什么

- 最终视觉页面或代码
- 可点击原型
- 详细字段、按钮与验收标准
- 生产数据、未确认能力或效果数据

## 7. 输入材料

### 7.1 最小输入

提供演示目标、方案主线或前置分析结果。

### 7.2 推荐输入

| 输入项 | 说明 |
|---|---|
| 演示对象 | 谁观看、关心什么 |
| 使用场景 | 讨论、评审或展示 |
| 业务主线 | 要展示的关键闭环 |
| 角色与页面 | 前置架构结论 |
| 制作约束 | 时长、工具和资源 |

## 8. 分析原则

先目标后页面；先业务路径后菜单；先核心角色后扩展角色；先首期能力后规划能力；样例数据必须明确为演示用途。

## 9. 分析流程

1. 识别对象、场景和目标。
2. 选择可闭环的主线。
3. 确定角色、端口和页面。
4. 设计路径与样例数据。
5. 标注 AI、模拟和人工复核边界。
6. 输出风险、任务和自检。

## 10. 标准输出结构

固定输出 15 个模块：输入理解、对象、目标、主线、端口、页面、路径、样例数据、页面重点、AI 边界、真实性边界、方案衔接、风险、制作建议和自检。

## 11. 与其他 Skill 的关系

05 使用 01 至 04 的结论规划 Demo；后续生成的页面、原型或演示文件由 06 进行一致性与风险质检。

## 12. 质量要求

- 主线可讲述并可走通
- 页面范围克制且可制作
- 样例数据可信且有标识
- AI 与模拟能力边界清楚
- 不以 Demo 代替生产验收

## 13. 输出风格要求

使用结构化标题和表格，区分核心展示、轻量展示、只讲不展示和暂不展示；未确认内容明确标记。

# 方案材料结构化 Skill

> 使用前请先对输入信息进行匿名化处理，不要粘贴客户机密、个人隐私、内部链接、账号密钥、未授权的项目原文或真实业务数据。示例中的组织、人物、项目和数据均为虚构或脱敏内容。

## 1. Skill 目标

把产品分析结果转译为可沟通、可汇报、可评审的材料结构，明确受众、目标、主线、章节、页级观点、图表建议和表达边界。

## 2. 关键边界声明

本 Skill 负责材料结构设计，不直接制作演示文稿、视觉稿、详细 PRD、技术方案或代码。

## 3. 使用前提

- 系统定位基本明确
- 业务流程和产品架构已有可引用结论
- 沟通对象、场景与预期动作可描述

## 4. 适用场景

- 方案沟通与决策汇报
- 立项、评审或阶段说明
- Demo 配套讲解
- 多方协作方案的结构化表达

## 5. 不适用场景

- 前置分析尚不清楚
- 直接进行视觉制作
- 详细需求或技术设计
- 无依据的包装性表达

## 6. 输入材料要求

### 6.1 最小输入

提供受众、目标或初步方案中的一种。

### 6.2 推荐输入

| 输入项 | 说明 |
|---|---|
| 受众与场景 | 谁看、在什么场合使用 |
| 预期动作 | 希望对方确认或决定什么 |
| 前置结论 | 定位、流程、架构与边界 |
| 事实依据 | 可验证的信息与来源 |
| 制作约束 | 时长、载体、页数或风格 |

### 6.3 可选补充信息

可提供已有目录、匿名草稿、图表思路或 Demo 规划。

## 7. 分析原则

先受众后结构；先目标后内容；先主线后材料页；先问题和价值后能力；先事实后表达；一页只表达一个核心观点。

## 8. 分析流程

1. 识别场景、受众与关注点。
2. 明确沟通目标和预期动作。
3. 提炼一句话主线。
4. 设计章节和页级结构。
5. 配置图表与 Demo 嵌入点。
6. 区分正文、附录、事实和待确认表达。

## 9. 标准输出结构

输出应包含：受众、沟通目标、方案主线、章节目录、页级结构、表达建议、图表配置、Demo 嵌入、版本建议、正文与附录、事实边界、风险和后续制作建议。

## 10. 与其他 Skill 的关系

04 引用 01 至 03 的结论形成材料结构，并为 05 的 Demo 规划提供讲述主线；成品材料可由 06 质检。

## 11. 使用注意事项

- 不重新进行前置分析。
- 不编造政策、成果或量化效果。
- 不让功能列表取代问题与价值主线。
- 重要表达需由责任人复核。

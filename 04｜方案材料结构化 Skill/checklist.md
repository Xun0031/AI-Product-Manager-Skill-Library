# 04｜方案材料结构化 Skill｜Checklist

> 使用前请先对输入信息进行匿名化处理，不要粘贴客户机密、个人隐私、内部链接、账号密钥、未授权的项目原文或真实业务数据。示例中的组织、人物、项目和数据均为虚构或脱敏内容。

## v1.1 方案表达专项自检

### 对象、目标与版本

- [ ] 已明确汇报对象、使用场景、已知信息、关注点和决策责任。
- [ ] 已明确材料目标及希望对方理解、确认、决定或执行的动作。
- [ ] 已选择高层决策、业务执行或对外沟通版本，并说明版本取舍。

### 信息分层与边界

- [ ] 事实、推断、建议和待确认项已分开，事实可追溯到来源类型。
- [ ] 对内可见、对外可用、需授权和禁止出现的内容已区分。
- [ ] 未将内部假设、未定责任、执行风险或未授权信息直接写入对外版本。
- [ ] 未编造主体、数据、政策依据、进展、效果或承诺。

### 章节与页级表达

- [ ] 章节顺序服务于材料目标，不是功能目录的简单堆叠。
- [ ] 每页只有一个核心观点，标题能够表达结论而非仅写主题。
- [ ] 每页均有支撑事实；缺少证据的页面已标记待补证或删除。
- [ ] 图表类型与核心观点匹配，不用装饰图替代论证。
- [ ] 讲述重点和正文信息互补，未依赖口头补充才能成立。

### 缺口、风险与行动

- [ ] 信息缺口说明了影响页面、确认对象和处理方式。
- [ ] 风险、边界、待确认事项和下一步动作均有责任归属。
- [ ] 高层版可直接支持决策，执行版可直接支持分工，对外版可安全沟通。

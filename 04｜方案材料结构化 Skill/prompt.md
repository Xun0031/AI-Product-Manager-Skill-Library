# 04｜方案材料结构化 Skill｜Prompt

> 使用前请先对输入信息进行匿名化处理，不要粘贴客户机密、个人隐私、内部链接、账号密钥、未授权的项目原文或真实业务数据。示例中的组织、人物、项目和数据均为虚构或脱敏内容。

使用时只填写匿名化后的必要信息。未知内容可留空，并要求 AI 标记为待确认。

先选择材料版本：高层决策简版、业务执行详版或对外沟通版。若同一任务需要多个版本，先建立共同事实底稿，再分别组织结构。

## 1. 标准调用提示词

### 使用场景

完整执行本 Skill 的标准分析流程，并对信息缺口单独标注。

### 输入

```text
【汇报对象与使用场景】
{{填写谁看、在什么场合使用、对方已知什么、最关注什么}}

【材料目标与预期动作】
{{填写希望对方理解、确认、决定、执行或对外转述什么}}

【材料版本】
{{高层决策简版 / 业务执行详版 / 对外沟通版 / 多版本}}

【已确认事实与来源类型】
{{填写可引用的定位、流程、架构、规则和验证信息}}

【推断、建议与待确认项】
{{分开填写，不要混入已确认事实}}

【对内 / 对外边界】
{{填写可公开、仅内部、需授权和禁止出现的内容}}

【约束与禁止事项】
{{填写时间、范围、表达边界和不能出现的内容}}
```

### 执行要求

1. 识别汇报对象、材料目标、预期动作和版本边界。
2. 将输入标为事实、推断、建议或待确认，事实注明来源类型和确认状态。
3. 先形成一句话主线和章节逻辑，再设计页级结构。
4. 每页只保留一个核心观点，并列支撑事实、图表建议、讲述重点和表达边界。
5. 高层版突出决策事项与关键取舍；执行版突出范围、流程、分工、计划与验收；对外版只使用已授权内容。
6. 信息不足时输出可成立的初步结构，并列出缺口、影响页面和确认对象。
7. 单列风险、边界、待确认事项和版本差异，不编造数据、结果、主体、政策依据或能力。

### 输出要求

- 使用清晰的 Markdown 标题、列表和必要表格。
- 先给结论，再给依据与细节。
- 输出事实 / 推断 / 建议分层表和版本策略。
- 页级表至少包含核心观点、支撑事实、图表建议、讲述重点、表达边界和信息缺口。
- 明确列出风险、待确认项和下一步。
- 结尾执行对应 checklist 自检。

## 1.1 三类材料版本差异

| 版本 | 核心任务 | 优先内容 | 应压缩或移除 |
|---|---|---|---|
| 高层决策简版 | 支持判断和取舍 | 决策事项、价值、关键方案、影响、风险、所需动作 | 操作步骤、字段细节、未形成结论的过程材料 |
| 业务执行详版 | 支持协作和落地 | 范围、流程、角色、任务、依赖、里程碑、验收、风险处置 | 与执行无关的长篇背景和宣传性表达 |
| 对外沟通版 | 建立共同理解 | 共同目标、场景、协作方式、已确认能力、双方责任、边界与下一步 | 内部取舍、未定责任、敏感假设、未经授权信息 |

## 2. 高层决策简版提示词

### 使用场景

围绕需要作出的决定组织材料，压缩过程细节，明确方案取舍、关键风险和所需动作。

### 输入

```text
【背景与目标】
{{填写匿名化背景、目标与输出用途}}

【已有材料】
{{填写已确认事实或前置 Skill 结论}}

【角色、流程或对象】
{{按当前任务填写；未知可留空}}

【约束与禁止事项】
{{填写时间、范围、表达边界和不能出现的内容}}
```

### 执行要求

1. 先区分已确认事实、合理推断和待确认项。
2. 按本 Skill 的标准流程处理，不跳过边界判断。
3. 信息不足时先给初步结果，再提出具体问题。
4. 首页或前置摘要明确“需要决定什么、为什么现在决定、不同选择的影响”。
5. 页级结构优先使用决策摘要、问题与影响、方案选择、关键闭环、范围边界、风险和决策事项。
6. 不编造数据、结果、主体、政策依据或能力。

### 输出要求

- 使用清晰的 Markdown 标题、列表和必要表格。
- 先给结论，再给依据与细节。
- 明确列出风险、待确认项和下一步。
- 结尾执行对应 checklist 自检。

# 04｜方案材料结构化 Skill｜公开版测试记录说明

> 公开版不保留原始测试记录，仅保留基于虚构输入的精简合成测试结果。

## 当前合成测试

> 本测试为合成测试，不含真实项目内容。

### v1.1｜同一项目三类材料结构

**测试目的**：验证 Skill 能否基于同一份虚构事实底稿，分别形成高层决策简版、业务执行详版和对外沟通版，并保持事实一致与表达边界。

**合成输入摘要**：虚构协同平台已有目标、角色和主流程等基础事实；首期范围属于建议，责任、时效、验收和授权范围待确认；需要为决策角色、执行角色和外部沟通对象准备不同材料结构。

**预期能力**：

1. 识别汇报对象、材料目标和预期动作；
2. 区分事实、推断、建议和待确认项；
3. 为三类版本形成不同章节与页级结构；
4. 每页给出核心观点、支撑事实、图表建议和表达边界；
5. 阻止内部假设和未授权信息进入对外版本。

**期望输出结构**：对象与目标 → 信息分层 → 内外边界 → 版本策略 → 方案主线 → 章节结构 → 页级结构 → 信息缺口 → 风险与待确认 → 自检。

**结果摘要**：合成演练能够让高层版围绕决策与取舍、执行版围绕责任与验收、对外版围绕共同目标与已授权边界展开。三个版本共享事实底稿，未将建议或待确认事项改写为既定事实。

**是否通过**：通过。核心结构满足 v1.1 预期；正式使用时仍需由材料责任人复核对外授权范围和最终口径。

## 保留结构的目的

- 保持与 Skill 库的测试组织方式一致；
- 为后续合成案例测试预留位置；
- 避免把未授权材料、可识别主体或项目过程带入公开仓库。

## 测试要求

1. 仅使用完全虚构或充分匿名化的输入；
2. 覆盖正向、边界和反向场景；
3. 记录目标、输入摘要、输出摘要、问题和结论；
4. 发布前再次执行敏感信息与阻断词检查。

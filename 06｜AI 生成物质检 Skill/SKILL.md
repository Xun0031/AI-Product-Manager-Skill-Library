# 06｜AI 生成物质检 Skill

> 使用前请先对输入信息进行匿名化处理，不要粘贴客户机密、个人隐私、内部链接、账号密钥、未授权的项目原文或真实业务数据。示例中的组织、人物、项目和数据均为虚构或脱敏内容。

## 1. Skill 定位

检查完整生成物是否符合目标、事实、结构和边界。默认只输出质检结论与返工建议，不直接重写整个生成物。

## 2. 核心价值

把“感觉不对”转化为可定位、可分级、可执行的问题清单，并在交付前形成质量闸口。

## 3. 适用场景

- 完整文档、演示稿或 Markdown 质检
- Demo、原型和静态页面质检
- 流程图、架构图、图表与看板质检
- 多文件材料包准入前检查

## 4. 不适用场景

- 原始目标或业务事实不清
- 专业代码、安全、性能或法务审查
- 生产系统验收

## 5. 质检对象范围

可检查文档、导出文件、页面截图、原型、静态演示、流程图、架构图、业务图表、演示脚本和材料包草稿。

## 6. 输入材料

### 6.1 最小输入

原始目标、生成物和使用场景。

### 6.2 推荐输入

| 输入项 | 说明 |
|---|---|
| 前置依据 | 01 至 05 的相关结论 |
| 目标受众 | 谁将使用生成物 |
| 验收标准 | 什么条件算可用 |
| 禁止事项 | 不能出现或承诺什么 |
| 交付阶段 | 草稿、评审或正式版本 |

## 7. 核心质检原则

先判断能不能读，再判断好不好；先识别类型，再启用分支；先看目标和事实，再看结构与表达；返工建议必须可执行。

## 8. 文件可读取性与质检限制判断

| 检查项 | 结论 |
|---|---|
| 文件是否可读取 | 【填写】 |
| 是否看到完整内容 | 【填写】 |
| 无法判断范围 | 【填写】 |
| 是否需要补充材料 | 【填写】 |

## 9. 质检分支选择机制

根据对象类型选择文档、演示稿、导出文件、Demo / 原型、流程图、架构图、图表或多文件综合分支。不相关分支标记为不适用。

## 10. 专项质检维度

文档关注结构、事实和可读性；Demo 关注路径、状态和角色边界；图表关注节点、关系、口径和观点支撑；多文件材料关注版本与准入。

## 11. 通用质量检查维度

固定关注目标、完整性、事实、定位、流程、架构、表达、页面、图表、数据、AI 能力、真实性和交付可用性。

## 12. 问题分级标准

| 等级 | 定义 |
|---|---|
| P0 | 阻塞，不能交付 |
| P1 | 必须修改后再交付 |
| P2 | 建议修改 |
| P3 | 可后续优化 |

## 13. 总体质检结论标准

| 结论 | 含义 |
|---|---|
| A | 通过 |
| B | 小修后可用 |
| C | 需要返工后复检 |
| D | 无法有效使用或建议重做 |

## 14. 标准输出结构

固定输出：输入理解、可读取性、范围、总体结论、依据映射、目标、结构、事实、角色流程架构、表达、页面图表、可用性、真实性边界、问题清单、返工建议、可选返工提示与自检。

## 15. 与其他 Skill 的关系

06 使用 01 至 05 的结论作为依据，并为 07 的材料准入提供质量状态。

## 16. 边界声明

不替代专业设计评审、代码审计、安全测试、性能测试、法务合规和生产验收。

## 17. 常用调用提示词设计

prompt.md 保留标准质检、文档、导出文件、Demo、图表、表达风险和返工提示等调用方式。

## 18. 示例案例设计

examples.md 使用完全虚构的文档、页面、图表与材料包案例，所有问题和数据均重新构造。

## 19. 正式文件结构

文件夹包含 SKILL.md、prompt.md、output-template.md、checklist.md、examples.md、changelog.md 和 test-results/。

## 20. 测试策略

公开版测试至少覆盖文档、Demo 和图表三类合成输入；不保留原始测试记录。

## 21. 测试通过标准

能识别对象与限制、选择正确分支、先给总体结论、发现事实和边界问题、输出分级清单与可执行返工建议，并避免越界。

- `examples.md`：查看虚构案例。
- `changelog.md`：查看公开版能力变化。

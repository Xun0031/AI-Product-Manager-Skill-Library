# 02｜业务流程梳理 Skill｜Checklist

> 使用前请先对输入信息进行匿名化处理，不要粘贴客户机密、个人隐私、内部链接、账号密钥、未授权的项目原文或真实业务数据。示例中的组织、人物、项目和数据均为虚构或脱敏内容。

## v1.1 专项自检

### 触发、输入与边界

- [ ] 已明确触发角色、触发事件、前置条件、起点、终点、取消和重开条件。
- [ ] 已逐类列出输入来源、载体、完整性要求和重复识别线索。
- [ ] 当前流程、目标流程、已确认事实、推断和待确认项未混写。

### 角色、节点与动作

- [ ] 每个关键节点均有一个明确的直接责任角色。
- [ ] 发起、执行、审批、确认、协作、知会和兜底角色已区分。
- [ ] 每个节点均包含输入、动作、判断、输出和下一状态。
- [ ] 人工动作与系统动作已分开，不把预期自动化能力写成现状。

### 状态、通知与异常

- [ ] 状态名称、进入条件、允许动作、退出条件和退回路径清晰。
- [ ] 通知对象、触发时机、提醒升级和审批责任已说明。
- [ ] 已检查信息缺失、重复、超时、退回、撤回、失败、异议和人工接管。
- [ ] 每条异常路径都有责任人、可恢复出口或明确终止条件。

### 反馈、数据与优化

- [ ] 已说明结果如何送达、确认、提出异议和归档。
- [ ] 数据记录点与业务目的对应，不直接下沉为数据库字段设计。
- [ ] 已识别无责任人、无回执、重复录入、状态不一致和异常无出口等断点。
- [ ] 每条优化建议对应具体断点，并说明前提、影响角色和待确认风险。

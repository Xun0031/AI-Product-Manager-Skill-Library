# 02｜业务流程梳理 Skill｜Examples

> 本案例为虚构案例，仅用于演示 Skill 使用方法。

以下案例的主体、角色、流程、名称和数据均为重新构造的教学内容。

## 示例 1：公共资源预约流程

### 场景背景

某虚构团队希望在“公共资源预约流程”场景中使用本 Skill。当前输入只有匿名化摘要，目标是形成可讨论、可确认的结构化结果。

### 输入摘要

- 目标：验证一条核心业务闭环。
- 角色：发起角色、处理角色、确认角色和观察角色。
- 约束：不使用生产数据，不接入正式系统，不作未确认承诺。
- 输出用途：教学演示与方法验证。

### 示例输出片段

主流程：发起 → 受理 → 处理 → 确认 → 归档。异常：信息不足、超时、退回和取消均保留原因与责任角色。

### 使用边界

- 不对应任何现实组织、人员或项目。
- 不沿用现有业务链路、数量、时间线或材料清单。
- 未确认内容统一标记为待确认。

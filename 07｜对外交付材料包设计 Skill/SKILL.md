# 07｜对外交付材料包设计 Skill

> 使用前请先对输入信息进行匿名化处理，不要粘贴客户机密、个人隐私、内部链接、账号密钥、未授权的项目原文或真实业务数据。示例中的组织、人物、项目和数据均为虚构或脱敏内容。

## 1. Skill 定位

将已完成并通过质检的材料组织为主次清晰、风险可控、便于接收方理解和行动的材料包。

## 2. “对外交付”的广义定义

指面向特定接收方进行正式材料交付，接收方可为外部合作方，也可为内部评审、业务确认或研发评估的责任团队。

## 3. 默认产出边界

默认输出材料包设计方案，不自动创建目录、移动文件、重命名、压缩或发送；执行文件操作需另行获得明确授权。

## 4. 适用场景

- 正式方案、评审或阶段说明
- 多份文档、图表与 Demo 的统一组织
- 需要区分主材料、辅助材料、附录和受控内容
- 交付前版本与风险检查

## 5. 不适用场景

- 单个生成物尚未质检
- 合同、法务或正式验收
- 直接重写文档或制作 Demo

## 6. 输入材料

至少提供接收方、交付目标和候选材料清单。建议补充版本、负责人、质检状态、授权范围、时间与发送方式。

## 7. 核心原则

先对象后材料；先目标后数量；先准入后分层；公开交付区与受控工作区分离；未确认内容不作为正式承诺。

## 8. 材料包类型

| 类型 | 适用场景 |
|---|---|
| 方案沟通包 | 解释问题、方案与路径 |
| 评审包 | 支撑判断与确认 |
| Demo 演示包 | 支撑讲解和演示 |
| 阶段说明包 | 汇总成果、问题与下一步 |

## 9. 标准工作流程

1. 识别接收方与目标。
2. 盘点材料和质检状态。
3. 判断准入与分层。
4. 设计目录、命名和版本。
5. 规划使用顺序。
6. 标注数据、演示和承诺边界。
7. 输出交付前清单。

## 10. 标准输出结构

完整输出包含 17 个模块：输入、对象、场景、目标、盘点、准入、策略、分层、主材料、辅助与附录、不建议交付、目录、命名版本、使用顺序、风险标注、检查清单和后续建议。

## 11. 轻量输出结构

快速场景可输出：场景判断、推荐材料、不建议材料、目录、命名版本、风险提醒和下一步。

## 12. 与 01～06 的关系

07 使用 01 至 05 的结论判断材料主线与边界，并使用 06 的质检结果决定材料是否准入。

## 13. 与真实交付 / 项目推进的边界

本 Skill 不替代合同、法务、发送审批、正式验收、项目计划管理或最终责任认定。

## 14. 质量标准

输出应能识别对象、场景、目标、材料状态、准入、分层、目录、版本、使用顺序和风险，并避免材料过多或未确认内容外发。

## 15. 文件设计

文件夹包含 SKILL.md、prompt.md、output-template.md、checklist.md、examples.md、changelog.md 和 test-results/。

## 16. 测试策略

公开版使用方案沟通、内部评审与 Demo 演示三类合成场景测试，不保留原始材料清单。

## 17. 通过标准

能正确分层、分离公开与受控内容、识别未质检或授权不明材料、给出清晰目录和检查清单，并保持操作边界。

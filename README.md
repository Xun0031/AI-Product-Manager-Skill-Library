# AI Product Manager Skill Library

面向 B 端产品经理的 AI Skill 库。本仓库为脱敏通用版 / 公开基础版，用于展示可复用的方法、模板与虚构案例。

公开版沉淀自实际工作方法，并已完成脱敏、通用化和结构镜像处理。公开内容不包含特定组织、人员、地域或项目资料。

## 1. Skill 库用途

本仓库用于把系统定位、业务流程、产品架构、方案材料、Demo 规划、生成物质检和材料交付等工作沉淀为可复用的结构化方法。

Skill 用于辅助分析、整理与检查，不能替代产品经理判断，也不能替代业务确认、技术评审、安全审查、合规审查或正式验收。

## 3. Quick Start

不要只输入一句“帮我梳理一下”。先提供最低有效输入，再让 Skill 标记信息缺口。

1. 根据任务选择 Skill，先阅读对应的 `SKILL.md`；
2. 按下面的最低有效输入模板准备匿名化信息；
3. 从 `prompt.md` 选择最接近的场景模板；
4. 用 `output-template.md` 约束结果结构；
5. 用 `checklist.md` 逐项复核，未确认内容不得写成事实。

### 3.1 最低有效输入模板

```text
【任务目标】希望解决什么问题，结果用于什么场景
【已知事实】已经确认的信息及其来源类型
【参与对象】涉及哪些角色、流程或材料受众
【范围边界】本次包含什么、不包含什么
【期望输出】希望得到的结构、深度与格式
【待确认项】当前未知、冲突或需要责任人确认的内容
【安全约束】已匿名化；不得补写真实主体、数据或未确认结论
```

### 3.2 好输入 / 差输入对比

| 类型 | 输入示例 | 主要问题 / 优点 |
|---|---|---|
| 差输入 | “帮我做一个方案。” | 缺少受众、目标、事实依据、范围和输出用途，结果容易退化为通用目录。 |
| 好输入 | “请基于以下匿名化事实，为业务负责人形成一份决策简版结构；目标是确认协作范围；区分事实、推断和建议；未知内容列为待确认；输出页级观点与图表建议。” | 受众、目标、事实边界和输出颗粒度明确，可形成可复核结果。 |

### 3.3 按任务选择 Skill

| 当前任务 | 建议 Skill | 进入条件 |
|---|---|---|
| 说清系统为什么存在、解决什么问题 | 01 系统定位 | 已有背景、目标或问题描述 |
| 还原多角色如何协作、异常如何处理 | 02 业务流程 | 已有流程描述、访谈摘要或操作记录 |
| 从角色出发确定工作台、能力和协作边界 | 03 产品架构 | 定位和主要流程基本明确 |
| 面向特定受众组织汇报或方案材料 | 04 方案结构 | 有可引用的定位、流程或架构结论 |
| 规划演示页面、路径和讲解重点 | 05 Demo 规划 | 方案主线和演示目标明确 |
| 检查 AI 生成文档、原型或图表 | 06 生成物质检 | 已有待检生成物及评价依据 |
| 组织正式交付材料包 | 07 交付材料包 | 交付对象、范围和准入要求明确 |
| 制作或迭代一个 Skill | 98 Skill 制作与迭代 | 有稳定、可重复的工作方法 |

### 3.4 多 Skill 组合示例

虚构任务：为一个匿名化的多方协作平台准备内部评审材料。

```text
01 明确平台目标与边界
→ 02 还原主流程、异常分支、状态和责任人
→ 03 将经确认的业务角色转译为产品架构
→ 04 形成面向评审对象的章节与页级结构
→ 06 检查事实边界、逻辑一致性和遗漏
```

组合使用时，后一 Skill 只引用前一 Skill 已确认的结论；推断与待确认项必须继续保留标签。

## 4. 每个 Skill 的文件结构

```text
XX｜某 Skill/
├── SKILL.md
├── prompt.md
├── output-template.md
├── checklist.md
├── examples.md
├── changelog.md
└── test-results/
    └── README.md
```

- `SKILL.md`：定位、边界、输入、流程和输出说明；
- `prompt.md`：按场景组织的可复制提示模板；
- `output-template.md`：标准输出结构；
- `checklist.md`：质量与边界检查；
- `examples.md`：虚构案例；
- `changelog.md`：公开版能力变化；
- `test-results/`：公开测试记录说明与后续合成测试。

## 5. 推荐使用顺序

```text
01 系统定位
→ 02 业务流程
→ 03 产品架构
→ 04 方案结构
→ 05 Demo 规划
→ 06 生成物质检
→ 07 交付材料包
```

98 为 Meta Skill，用于制作和迭代 Skill，不属于 01 至 07 的业务链路。

## 6. 使用方法

1. 根据当前问题选择 Skill 文件夹；
2. 阅读 `SKILL.md`，确认适用场景和边界；
3. 从 `prompt.md` 选择调用模板并填写匿名化输入；
4. 使用 `output-template.md` 组织结果；
5. 使用 `checklist.md` 自检；
6. 需要示范时阅读 `examples.md`，不要把案例数据当成事实。

Skill 不能替代产品经理判断。对业务边界、优先级、可行性、责任归属和正式对外口径，仍需由相应责任人确认。

## 7. 隐私与安全

不要向 AI 输入客户机密、个人隐私、内部链接、账号密钥或未授权项目原文。提交前删除可识别名称、精确地址、联系方式、访问凭证、本地路径和隐藏元数据。

所有公开案例均为虚构或脱敏案例，公开测试均为合成测试，不代表任何现实组织、人员或项目。

## 8. 公开版维护说明

- 保持原库的 Skill 文件夹与主要 Markdown 文件组织方式；
- 原始案例、测试记录和历史过程不进入公开版；
- 高风险内容使用合成案例或公开占位说明；
- 每次更新后执行文件类型、敏感信息和阻断词复检。
